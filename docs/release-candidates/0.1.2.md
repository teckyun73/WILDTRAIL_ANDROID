# WildTrail Android RC 0.1.2 (3)

This release candidate includes the audio identification client safeguards and model metadata visibility updates prepared after RC 0.1.1.

## Version

| Field | Value |
| --- | --- |
| Version code | `3` |
| Version name | `0.1.2` |
| Package | `com.wildtrail.app` |
| Workflow source | `.github/workflows/android-ci.yml` |

## Scope

- Blocks saving audio identification candidates that are not present in the current species catalog.
- Shows model metadata and class-count diagnostics on the status screen.
- Adds a client-side 20MB audio upload limit before sending requests to the backend.
- Handles HTTP 413 file-size failures with a user-facing audio size message.
